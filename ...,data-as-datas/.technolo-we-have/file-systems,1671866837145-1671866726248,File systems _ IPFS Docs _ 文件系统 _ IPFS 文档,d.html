# File systems and IPFS# 文件系统和IPFS

Working with files in IPFS can be a little different than you're used to for several reasons:由于一些原因，在IPFS中处理文件可能与你：

  • Content addressing means that when files change, the content identifier (CID) of those files changes too.内容寻址意味着当文件改变时，这些文件的内容标识符（CID）也会改变。
  • Files may be too big to fit in a single block, so IPFS splits the data into multiple blocks and uses metadata to link it all together.文件可能太大，无法容纳在一个区块中，所以IPFS将数据分成多个区块，并使用元数据将其连接在一起。

Mutable File System (MFS) and Unix File System (UnixFS) can help you address these new ways of thinking of files.可变文件系统（MFS）和Unix文件系统（UnixFS）可以帮助你解决这些对文件的新思考方式。

# Mutable File System (MFS)# 可变文件系统（MFS）

Because files in IPFS are content-addressed and immutable, they can be complicated to edit. Mutable File System (MFS) is a tool built into IPFS that lets you treat files like you would a regular name-based filesystem — you can add, remove, move, and edit MFS files and have all the work of updating links and hashes taken care of for you.由于IPFS中的文件是由内容寻址的，而且是不可改变的，因此编辑起来可能很复杂。可变文件系统（MFS）是一个内置于IPFS中的工具，它可以让你像对待普通的基于名字的文件系统一样对待文件--你可以添加、删除、移动和编辑MFS文件，并为你处理所有更新链接和散列的工作。

# Working with files API# 使用文件API工作

MFS is accessed through the files commands in the IPFS CLI and API. The commands on the files API take the format of ipfs.files.someMethod(). These methods are used to:MFS是通过IPFS CLI和API中的files命令访问的。files API上的命令采用ipfs.files.someMethod()的格式。这些方法被用来：

  • Create a directory创建一个目录
  • Check directory status检查目录状态
  • Add a file to MFS向MFS添加一个文件
  • View contents of a directory查看一个目录的内容
  • Copy a file or directory复制一个文件或目录
  • Move a file or directory移动一个文件或目录
  • Read the contents of a file读取一个文件的内容
  • Remove a file or directory删除一个文件或目录

Prefer hands-on learning? Explore these MFS methods in ProtoSchool's Mutable File System (opens new window) tutorial, solving coding challenges right in your web browser.更喜欢实践学习？在ProtoSchool的Mutable File System中探索这些MFS方法。 (打开新窗口) 教程，在你的网络浏览器中解决编码难题。

# Create a directory# 创建一个目录

The MFS method ipfs.files.mkdir creates a new directory at a specified path. For example, to add a directory example to our root directory (/), run:MFS方法ipfs.files.mkdir在指定的路径上创建一个新目录。例如，要在我们的根目录 (/) 下添加一个目录实例，运行：

await ipfs.files.mkdir('/example')

If you want to create a new directory nested under others that don't yet exist, you need to explicitly set the value of parents to true, like so:如果你想创建一个嵌套在其他尚不存在的目录下的新目录，你需要明确地将parent的值设置为true，像这样：

await ipfs.files.mkdir('/my/directory/example', { parents: true })

# Check directory status# 检查目录状态

The method, ipfs.files.stat enables you to check the status of a file or directory on your IPFS node. To check the status of a directory called example located within the root directory, you could call the method by running:方法，ipfs.files.stat使你能够检查你的IPFS节点上的文件或目录的状态。要检查位于根目录中的名为example的目录的状态，你可以通过运行该方法调用：

await ipfs.files.stat('/example')

This method returns an object with a cid, size, cumulativeSize, type, blocks, withLocality, local, and sizeLocal.该方法返回一个带有cid、size、cumulativeSize、type、block、withLocality、local和sizeLocal的对象。

// {
//   hash: CID('QmXmJBmnYqXVuicUfn9uDCC8kxCEEzQpsAbeq1iJvLAmVs'),
//   size: 60,
//   cumulativeSize: 118,
//   blocks: 1,
//   type: 'directory'
// }

If you add, move, copy, or remove a file into a directory, the hash of that directory will change with every file modified.如果你在一个目录中添加、移动、复制或删除一个文件，该目录的哈希值将随着每个文件的修改而改变。

# Add a file to MFS# 添加一个文件到MFS

To add a file to IPFS, use the MFS ipfs.files.write method using the command:要添加一个文件到IPFS，使用MFS ipfs.files.write方法，使用命令：

await ipfs.files.write(path, content, [options])

TIPTIP

This method can create a brand new file that accepts file content in multiple formats, in a specified path within the IPFS instance by providing the boolean option {create: true }.这个方法可以创建一个全新的文件，接受多种格式的文件内容，通过提供布尔选项{create: true }在IPFS实例中的指定路径。

To add a file object called examplePic to the root directory you could run:要在根目录下添加一个名为examplePic的文件对象，你可以运行：

await ipfs.files.write('/example.jpg', examplePic, { create: true })

This method does not provide a return value.这个方法不提供返回值。

# View contents of a directory# 查看一个目录的内容

To check whether the ipfs.files.write method has worked as expected, use the ipfs.files.ls method to inspect directories on the MFS by running:为了检查ipfs.files.write方法是否按预期工作，使用ipfs.files.ls方法通过运行来检查MFS上的目录：

await ipfs.files.ls([path], [options])

The method will default to listing the contents of your directory (/), or you can choose to specify a specific path you'd like to inspect:该方法将默认列出你的目录（/）的内容，或者你可以选择指定一个你想检查的特定路径：

await ipfs.files.ls('/example')

This method produces an array of objects for each file or directory with properties such as name, type, size, cid, mode, and mtime. If you wanted to inspect the contents of a /example directory, run:这个方法为每个文件或目录产生一个对象数组，其属性包括名称、类型、大小、cid、模式和mtime。如果你想检查一个/example目录的内容，运行：

await ipfs.files.ls('/example')

# Copy a file or directory# 复制一个文件或目录

In the Mutable File System, like traditional systems, you can copy a file or directory to a new location while also leaving it intact at its source.在可变文件系统中，像传统的系统一样，你可以把一个文件或目录复制到一个新的位置，同时也把它完整地保留在它的来源处。

You can do this by using the method:你可以通过使用方法来做到这一点：

await ipfs.files.cp(...from, to, [options])

TIPTIP

This method offers two formatting options for passing the from key:这个方法为传递from键提供了两个格式化选项：

  • an existing MFS path to a file or a directory in your node (e.g. /my-example-dir/my-example-file.txt)一个现有的MFS路径，指向你的节点中的一个文件或一个目录（例如：/my-example-dir/my-example-file.txt）。
  • an IPFS path to a file or directory hosted either by you or by a peer (e.g. /ipfs/QmWc7U4qGeRAEgtsyVyeW2CRVbkHW31nb24jFyks7eA2mF)一个IPFS路径，指向一个由你或同伴托管的文件或目录（例如：/ipfs/QmWc7U4qGeRAEgtsyVyeW2CRVbkHW31nb24jFyks7eA2mF)

The to key is the destination path in MFS, and there's an option {create: true } that can be used to create parent directories that don't already exist.to键是MFS中的目标路径，有一个选项{create: true }，可以用来创建不存在的父目录。

You can use this method to perform different operations including:你可以使用这个方法来执行不同的操作，包括：

// copy a single file into a directory
await ipfs.files.cp('/example-file.txt', '/destination-directory')
await ipfs.files.cp('/ipfs/QmWGeRAEgtsHW3ec7U4qW2CyVy7eA2mFRVbk1nb24jFyks', '/destination-directory')

// copy multiple files into a directory
await ipfs.files.cp('/example-file-1.txt', '/example-file-2.txt', '/destination-directory')
await ipfs.files.cp('/ipfs/QmWGeRAEgtsHW3ec7U4qW2CyVy7eA2mFRVbk1nb24jFyks',
 '/ipfs/QmWGeRAEgtsHW3jk7U4qW2CyVy7eA2mFRVbk1nb24jFyre', '/destination-directory')

// copy a directory into another directory
await ipfs.files.cp('/source-directory', '/destination-directory')
await ipfs.files.cp('/ipfs/QmWGeRAEgtsHW3ec7U4qW2CyVy7eA2mFRVbk1nb24jFyks', '/destination-directory')

# Move a file or directory# 移动一个文件或目录

MFS allows you to move files between directories using the ipfs.files.mv, which looks like this:MFS允许你使用ipfs.files.mv在目录之间移动文件，它看起来像这样：

await ipfs.files.mv(from, to, [options])

from is the source path (or paths) of the content you'd like to move, while to is the destination path.from是你想移动的内容的源路径（或路径），而to是目标路径。

You can use this method to perform different operations including:你可以使用这个方法来执行不同的操作，包括：

// move a single file into a directory
await ipfs.files.mv('/example-file.txt', '/destination-directory')

// move a directory into another directory
await ipfs.files.mv('/source-directory', '/destination-directory')

// overwrite the contents of a destination file with the contents of a source file
await ipfs.files.mv('/source-file.txt', '/destination-file.txt')

// move multiple files into a directory
await ipfs.files.mv('/example-file-1.txt', '/example-file-2.txt', '/example-file-3.txt', '/destination-directory')

# Read the contents of a file# 读取一个文件的内容

The ipfs.files.read method allows you to read and, or display the contents of a file in a buffer. The method takes the format:ipfs.files.read方法允许你读取和，或显示缓冲区中的文件内容。该方法的格式是：

ipfs.files.read(path, [options])

The path provided is the path of the file to read, and it must point to a file rather than a directory.提供的路径是要读取的文件的路径，它必须指向一个文件而不是一个目录。

# Remove a file or directory# 删除一个文件或目录

MFS allows you to remove files or directories using the method:MFS允许你使用该方法来删除文件或目录：

await ipfs.files.rm(...paths, [options])

paths are one or more paths to remove.paths是一个或多个要删除的路径。

By default, if you attempt to remove a directory that still has contents, the request will fail. To remove a directory and everything contained in it, you'll need to use the option recursive: true.默认情况下，如果你试图删除一个仍有内容的目录，该请求将失败。要删除一个目录和其中包含的所有内容，你需要使用选项递归：true。

// remove a file
await ipfs.files.rm('/my/beautiful/file.txt')

// remove multiple files
await ipfs.files.rm('/my/beautiful/file.txt', '/my/other/file.txt')

// remove a directory and its contents
await ipfs.files.rm('/my/beautiful/directory', { recursive: true })

// remove a directory only if it is empty
await ipfs.files.rm('/my/beautiful/directory')

# Unix File System (UnixFS)# Unix文件系统(UnixFS)

When you add a file to IPFS, it might be too big to fit in a single block, so it needs metadata to link all its blocks together. UnixFS is a protocol-buffers (opens new window)-based format for describing files, directories, and symlinks in IPFS. This data format is used to represent files and all their links and metadata in IPFS. UnixFS creates a block (or a tree of blocks) of linked objects.当你向IPFS添加一个文件时，它可能太大，不能装在一个块中，所以它需要元数据来把它的所有块连接起来。UnixFS是一个协议缓冲区 (打开新窗口) -的格式，用于描述IPFS中的文件、目录和符号链接。这种数据格式被用来表示文件和它们在IPFS中的所有链接和元数据。UnixFS创建了一个链接对象的块（或块的树）。

UnixFS currently has Javascript (opens new window) and Go (opens new window) implementations. These implementations have modules written in to run different functions:UnixFS目前有Javascript (打开新窗口) 和Go (打开新窗口) 实现。这些实现都有写进的模块来运行不同的功能：

  • Data Formats: manage the serialization/deserialization of UnixFS objects to protocol buffers数据格式：管理UnixFS对象对协议缓冲区的序列化/反序列化

  • Importer: Build DAGs from files and directories输入器：从文件和目录建立DAGs

  • Exporter: Export DAGs输出器：输出DAGs

# Data Formats 数据格式

On UnixFS-v1 the data format is represented by this protobuf:在UnixFS-v1上，数据格式由这个protobuf表示：

message Data {
    enum DataType {
        Raw = 0;
        Directory = 1;
        File = 2;
        Metadata = 3;
        Symlink = 4;
        HAMTShard = 5;
    }

    required DataType Type = 1;
    optional bytes Data = 2;
    optional uint64 filesize = 3;
    repeated uint64 blocksizes = 4;
    optional uint64 hashType = 5;
    optional uint64 fanout = 6;
    optional uint32 mode = 7;
    optional UnixTime mtime = 8;
}

message Metadata {
    optional string MimeType = 1;
}

message UnixTime {
    required int64 Seconds = 1;
    optional fixed32 FractionalNanoseconds = 2;
}

This Data object is used for all non-leaf nodes in UnixFS:这个数据对象用于UnixFS的所有非叶子节点：

  • For files that are comprised of more than a single block, the Type field will be set to File, the filesize field will be set to the total number of bytes in the files, and blocksizes will contain a list of the filesizes of each child node.对于由一个以上的块组成的文件，类型字段将被设置为文件，文件大小字段将被设置为文件的总字节数，块大小将包含每个子节点的文件大小列表。

  • For files comprised of a single block, the Type field will be set to File, filesize will be set to the total number of bytes in the file, and file data will be stored in the Data field.对于由单个块组成的文件，类型字段将被设置为文件，文件大小将被设置为文件中的总字节数，而文件数据将被存储在数据字段中。

UnixFS also supports two optional metadata format fields:UnixFS还支持两个可选的元数据格式字段：

  • mode - used for persisting the file permissions in numeric notation (opens new window). If unspecified, this field defaults to 0755 for directories/HAMT shards and 0644 for all the other types where applicable.mode - 用于以数字符号保存文件的权限 (打开新窗口) . 如果没有指定，这个字段对于目录/HAMT碎片默认为0755，对于所有其他适用的类型默认为0644。

  • mtime - is a two-element structure (Seconds, FractionalNanoseconds) representing the modification time in seconds relative to the Unix epoch 1970-01-01T00:00:00Z.mtime - 是一个双元素结构(Seconds, FractionalNanoseconds)，代表相对于Unix纪元1970-01-01T00:00:00Z的修改时间，以秒为单位。

# Importer# 导入器

Importing a file into UnixFS is split into two processes. A chunking function and a layout function. You can test these features using the IPFS DAG builder (opens new window).将一个文件导入UnixFS分为两个过程。一个分块功能和一个布局功能。你可以使用IPFS DAG生成器来测试这些功能 (打开新窗口) .

# Chunking# 分块

When an object is added to IPFS, it is chunked up into smaller parts, each part is hashed, and a CID is created for each chunk. This DAG building process has two main parameters, the leaf format and the chunking strategy.当一个对象被添加到IPFS时，它被分割成更小的部分，每个部分都被散列，并且为每个分块创建一个CID。这个DAG构建过程有两个主要参数，叶子格式和分块策略。

The leaf format takes two format options, UnixFS leaves and raw leaves:叶子格式有两个格式选项，UnixFS叶子和原始叶子：

  • The UnixFS leaves format adds a data wrapper on newly added objects to produce UnixFS leaves with additional data sizes. This wrapper is used to determine whether newly added objects are files or directories. This format is the default for CIDv0.UnixFS叶子格式在新添加的对象上添加一个数据包装器，以产生具有额外数据大小的UnixFS叶子。这个包装器被用来确定新添加的对象是文件还是目录。这种格式是CIDv0的默认格式。

  • The raw leaves format on IPFS where nodes output from chunking will be raw data from the file with a CID codec of 'raw'. This is mainly configured for backward compatibility with formats that used a UnixFS Data object. This format is the default for CIDv1 created with ipfs add --cid-version 1, soon to become the global default.IPFS上的原始叶子格式，其中分块输出的节点将是来自文件的原始数据，CID编解码为 "raw"。这主要是为了向后兼容使用UnixFS数据对象的格式而配置的。这种格式是用ipfs add --cid-version 1创建的CIDv1的默认格式，很快将成为全球默认格式。

The chunking strategy is used to determine the size options available during the chunking process. The strategy currently has two different options, 'fixed size' and 'rabin'.分块策略用于确定在分块过程中可用的大小选项。该策略目前有两个不同的选项，"固定大小 "和 "rabin"。

  • Fixed sizing will chunk the input data into pieces of a given size. This could be 512 bytes, 1024 bytes, and more—the smaller the byte size, the better the deduplication of data.固定大小将把输入的数据分块成一个给定的大小。这可能是512字节，1024字节，甚至更多--字节大小越小，重复数据删除的效果越好。

  • Rabin chunking will chunk the input data using Rabin fingerprinting to determine the boundaries between chunks. Rabin also reduces the number of input data chunked nodes.拉宾分块将使用拉宾指纹对输入数据进行分块，以确定分块之间的边界。Rabin也会减少输入数据分块的节点数量。

# Layout# 布局

The layout defines the shape of the tree that gets built from the chunks of the input file.布局定义了从输入文件的各块中建立的树的形状。

There are currently two options for layout, balanced and trickle. Additionally, a max-width must be specified. The default max width is 174.目前有两个布局选项，平衡和涓流。此外，必须指定一个最大宽度。默认的最大宽度是174。

The balanced layout creates a balanced tree of width max-width. The tree is formed by taking up to max-width chunks from the chunk stream and creating a UnixFS file node that links to all of them. This is repeated until max-width UnixFS file nodes are created, at which point a UnixFS file node is created to hold all of those nodes recursively. The root node of the resultant tree is returned as the handle to the newly imported file.平衡式布局创建一个最大宽度的平衡树。该树是通过从块流中抽取最大宽度的块，并创建一个链接到所有这些块的UnixFS文件节点来形成的。这样反复进行，直到创建了最大宽度的UnixFS文件节点，这时再创建一个UnixFS文件节点来递归地容纳所有的这些节点。结果树的根节点被返回作为新导入文件的句柄。

If there is only a single chunk, no intermediate UnixFS file nodes are created, and the single chunk is returned as the handle to the file.如果只有一个大块，则不创建中间的UnixFS文件节点，并将单个大块作为文件的句柄返回。

# Exporter# 导出器

To export or read the file data out of the UnixFS graph, perform an in-order traversal, emitting the data contained in each of the leaves.要从UnixFS图中导出或读出文件数据，要执行一个顺序的遍历，发出每个叶子中包含的数据。
